=========================================================
КАК РАБОТАЕТ БОТ
=========================================================

Стандартный сценарий конкурса:

1. Пользователь открывает бота.

2. Бот показывает описание конкурса.

3. Пользователь нажимает кнопку перехода на пресейв.

4. Делает пресейв трека.

5. Делает скриншот.

6. Возвращается в бот.

7. Нажимает кнопку "Отправить скриншот".

8. Отправляет фото ИЛИ изображение документом.

9. Бот:

    • сохраняет изображение
    • уменьшает его без потери читаемости
    • записывает пользователя в PostgreSQL
    • благодарит пользователя

После этого пользователь считается зарегистрированным.

=========================================================
РОЗЫГРЫШ
=========================================================

Количество победителей задается в config.json.

Розыгрыш производится случайным образом
из зарегистрированных участников.

Победители выбираются через PostgreSQL:

ORDER BY RANDOM()

LIMIT N

=========================================================
РАССЫЛКА
=========================================================

Администратор может отправить сообщение всем участникам.

Рассылка используется после окончания конкурса
или для объявления результатов.

=========================================================
ПРОВЕРКА ПОДПИСКИ
=========================================================

Сейчас отключена.

Если понадобится вернуть:

1. Включить

check_subscription=true

2. Добавить username канала

3. Выдать боту права администратора
в проверяемом канале.

=========================================================
ПОВТОРНАЯ РЕГИСТРАЦИЯ
=========================================================

Один Telegram ID может участвовать только один раз.

Повторная отправка изображения
не записывается в БД.
